**IMPORTANT PATTERNS:**

**For buttons:**
```javascript
import getButtonVariants from './button-variants.js';
import { applyButtonVariant } from '../../scripts/theme-utils.js';

const variants = getButtonVariants();
const btn = document.createElement('a');
btn.className = 'button'; // NOT 'button acc-button--link'
const buttonElement = applyButtonVariant(btn, variants[variantName]);
container.append(buttonElement); // Append wrapper, not button
```

**For separators:**
```javascript
import getSeparatorVariants from './separator-variants.js';
import { applySeparatorVariant } from '../../scripts/theme-utils.js';

const separators = getSeparatorVariants();
const hr = document.createElement('hr');
hr.className = 'separator';
const separatorElement = applySeparatorVariant(hr, separators[variantName]);
container.append(separatorElement);
```

**For typography:**
```javascript
import getTypographyVariants from './typography-variants.js';
import { applyTypographyStyle } from '../../scripts/theme-utils.js';

const typography = getTypographyVariants();
const title = document.createElement('h2');
applyTypographyStyle(title, typography[styleName]); // Direct class application
container.append(title);
```

**Model field naming:**
- Buttons: `cta{N}Link`, `cta{N}Variant` (NO `cta{N}Text` field!)
- Separators: `separator{N}SeparatorVariant`
- Typography: `title{N}TypographyStyle`

**Row destructuring:**
```javascript
// CORRECT - NO separate text rows for buttons
const [
  titleRow,
  cta1LinkRow,     // Contains link AND text
  cta1VariantRow,  // Next row is variant
  cta2LinkRow,
  cta2VariantRow,
] = rows;

// WRONG - DO NOT DO THIS
const [
  titleRow,
  cta1LinkRow,
  cta1TextRow,     // ❌ DOESN'T EXIST
  cta1VariantRow,
] = rows;
```

## CRITICAL RULES

1. ❌ **NEVER** modify `theme.css` directly (it's auto-generated by AEM UI Solution)
2. ❌ **NEVER** create separate link text fields (`cta{N}Text` or `cta{N}LinkText`)
3. ❌ **NEVER** add `acc-button--link` class manually - `applyButtonVariant()` does it
4. ✅ **ALWAYS** call `moveInstrumentation()` BEFORE `applyButtonVariant()`
5. ✅ **ALWAYS** append the WRAPPER returned by theme applier functions, not the element itself
6. ✅ **ALWAYS** add component to `models/_section.json` filters (Step 9)
7. ✅ **ALWAYS** run `npm run build:themes` after creating model
8. ✅ **ALWAYS** run `npm run lint` before finishing
9. ✅ **ALWAYS** use TodoWrite tool to track progress

## THEME SYSTEM ARCHITECTURE

```
AEM UI Solution Platform
         ↓
    theme.css (auto-generated, READ ONLY)
         ↓
generate-theme-variants.js (extracts all themes)
         ↓
    ┌──────────┬──────────────┬──────────────┐
    ↓          ↓              ↓              ↓
button-    separator-   typography-     (future themes)
variants.js variants.js  variants.js
    ↓          ↓              ↓
theme-utils.js (applier functions)
    ↓          ↓              ↓
    applyButtonVariant
    applySeparatorVariant
    applyTypographyStyle
    applyThemeVariant (generic)
```
